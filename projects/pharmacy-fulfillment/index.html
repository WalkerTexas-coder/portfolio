Pharmacy Fulfillment Workflow

Product Case Study ‚Ä¢ Teligant Platform

The Problem: Prescription Routing Chaos

Doctors were manually routing prescriptions across 4 pharmacy partners (Crafted RX, Greenwich/Texas Performance Meds, Outreach RX, patient local pharmacy) with no systematic guidance, risking missed and duplicate prescriptions.

The Core Issue

The real issue wasn't just unclear routing logic‚Äîdoctors had no audit trail to track what they'd sent. Without visibility into previous routing decisions, they couldn't prevent errors or learn from patterns.

Product Lifecycle

Phase 1: Problem Discovery (Ideation)

I spent two days shadowing doctors to understand their workflow. Through observation and interviews, I discovered:

  • Doctors switching between multiple pharmacy partner systems
  • No clear rules for which pharmacy handles which medication
  • Duplicate prescriptions sent to multiple pharmacies
  • No audit trail of routing decisions
  • Manual workarounds creating operational risk

Phase 2: User Research & Requirements (Validation)

After identifying the problem, I validated the requirements with stakeholders:

  • Doctors needed clear routing guidance at prescription time
  • System must prevent duplicate routing
  • Complete audit trail for compliance and troubleshooting
  • Solution must integrate with existing DoseSpot e-prescribing iframe
  • Fast implementation‚Äîcouldn't wait months for complex automation

Phase 3: Solution Design (Implementation)

Product judgment: Built manual workflow bridge above DoseSpot iframe rather than complex automation, balancing speed with reliability.

This decision prioritized:

  • Fast delivery (3 weeks vs. 3 months for full automation)
  • Reliability (simple UI with clear guidance)
  • Audit trail (every routing decision tracked)
  • Flexibility (easy to adjust rules as pharmacy partnerships evolve)

Phase 4: Development & Delivery

Holistic implementation: Wrote 12 detailed Jira tickets across 2 sprint cycles with acceptance criteria and edge cases.

Implementation Scope

  • UI component above DoseSpot iframe with routing logic
  • Database schema for audit trail
  • Pharmacy partner selection rules engine
  • Admin interface for managing routing rules
  • Error handling and validation
  • Analytics and reporting dashboard

Phase 5: Testing & Validation

Worked directly with doctors during pilot rollout:

  • Beta tested with 2 doctors for 1 week
  • Gathered feedback on UI clarity and routing logic
  • Adjusted pharmacy selection rules based on real usage
  • Verified audit trail captured all necessary data

Phase 6: Deployment & Scale

Full rollout to all doctors within 3 weeks of initial deployment.
