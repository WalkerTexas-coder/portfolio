Platform Architecture

Multi-Tenant Design

Defined multi-tenant SaaS architecture enabling multiple healthcare brands on a single platform:

  • <strong>Isolated Tenant Instances:</strong> Each brand receives isolated patient portal, doctor portal, admin dashboard, and API endpoints
  • <strong>AWS ECS Deployment:</strong> Separate Docker containers for each service layer ensuring scalability
  • <strong>Cross-Tenant Data Transfer:</strong> Built HIPAA-compliant patient data transfer with explicit consent workflow
  • <strong>Rapid White-Label Deployment:</strong> Architecture enables quick brand launches with isolated data

Core Technical Integrations

Coordinated 8+ third-party API integrations with complex compliance requirements:

  • <strong>DoseSpot E-Prescribing:</strong> HIPAA-compliant electronic prescriptions for non-controlled substances, 514+ medication protocols with dosage variants
  • <strong>SureScripts Network:</strong> Pharmacy transmission connectivity for multi-state prescription fulfillment
  • <strong>Choose Health Labs:</strong> 7 REST endpoints plus webhook system for at-home lab testing with real-time results
  • <strong>PayTheory Billing:</strong> Flexible subscription processor supporting 30-365 day billing cycles
  • <strong>ShipStation Fulfillment:</strong> Order management and shipping coordination via Custom Store integration
  • <strong>Claude AI (AWS Bedrock):</strong> Clinical documentation automation with HIPAA BAA and zero retention
