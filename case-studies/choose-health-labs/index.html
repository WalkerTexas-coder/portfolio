The Problem: Limited Patient Lifetime Value

Hedfirst was migraine-only, limiting patient lifetime value to $499/year with no expansion revenue. We needed a way to increase engagement and create additional value for patients beyond single-condition treatment.

Product Lifecycle

Phase 1: Strategic Hypothesis (Ideation)

Hypothesis: Migraine patients would value comprehensive health monitoring, especially if lab results revealed underlying conditions (vitamin deficiencies, hormone imbalances) contributing to migraines.

Value Clarification

This wasn't just feature expansion‚Äîit positioned Hedfirst as comprehensive health partner vs. single-condition treatment, increasing LTV and patient engagement while improving clinical outcomes.

Phase 2: Vendor Evaluation & Requirements (Validation)

Evaluated multiple at-home lab testing providers based on:

  • API quality and documentation
  • HIPAA compliance and BAA availability
  • Test panel options relevant to migraine care
  • Pricing structure for wholesale partnerships
  • Technical integration complexity

Selected Choose Health despite higher cost due to superior API documentation and webhook system‚Äîestimated this decision would save 2+ weeks of integration time.

Phase 3: API Design & Prototyping (Implementation)

I didn't just hand this to engineers. I analyzed the 7-endpoint Choose Health API, mapped the JSON response fields to our internal data model, and designed the webhook handler for asynchronous results delivery. I wrote the Swagger documentation for our internal developers to consume the external API wrapper.

  • <strong>Order Creation:</strong> POST endpoint for creating lab kit orders
  • <strong>Order Status:</strong> GET endpoint for tracking fulfillment
  • <strong>Test Catalog:</strong> Available lab panels and pricing
  • <strong>Results Webhook:</strong> Real-time results delivery when labs complete
  • <strong>PDF Generation:</strong> Formatted results for provider review
  • <strong>Patient Portal:</strong> Iframe integration for requisition forms
  • <strong>Billing Integration:</strong> Webhook for payment confirmation

Designed patient-facing lab ordering flow and provider dashboard for results review with interactive React prototypes.

Phase 4: Development & Delivery

Holistic implementation: Wrote 13 implementation tickets across frontend, backend, admin interfaces, and HIPAA-compliant data storage with 7-year retention.

  • Patient portal: Lab ordering, requisition forms, results viewing
  • Provider dashboard: Results review, clinical notes, lab history
  • Admin interface: Order management, fulfillment tracking
  • Backend: 7 API endpoints, webhook handlers, data models
  • Compliance: HIPAA-compliant storage, audit logging, 7-year retention
  • Billing: Integration with PayTheory for lab kit charges

Phase 5: Beta Testing & Iteration

Launched to beta patients with close monitoring:

  • 25 beta patients offered lab testing in month 1
  • Gathered feedback on ordering flow and results presentation
  • Refined provider dashboard based on doctor usage patterns
  • Adjusted lab panel recommendations based on clinical findings

Phase 6: Scale & Clinical Integration

Full rollout with clinical workflow integration:

  • Lab recommendations built into consultation flow
  • 15-minute provider review sessions included with lab purchases
  • Follow-up protocols for abnormal results
  • Treatment adjustments based on lab insights
